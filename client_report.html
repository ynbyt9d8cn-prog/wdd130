Client Report

How did you decide what content and images to use? I used the site plan to choose content that clearly explained the purpose of the site and supported the theme. I selected images that matched the topic and helped the pages feel complete and consistent with the wireframe.

How did you communicate with your developer? Any issues? I communicated by sharing the site plan, links, and feedback as pages were published. Communication was mostly smooth, and when issues came up, they were usually related to waiting for files to be pushed before work could continue.

Did your developer meet your expectations? Yes. The homepage was published correctly and updated with CSS styling to match the site plan and wireframe.

What will you do differently next time? Next time I would set clearer deadlines and check-in times, and I would confirm file structure (CSS file name, image folder, link paths) earlier so fewer small issues slow down progress.

Lead Developer Report

How did you manage your team and communicate tasks? I shared the repository, provided the expected page assignments, and made sure junior developers knew to use the same stylesheet and pull changes before editing.

How did you communicate with your client? Any issues? I sent the published site link, asked for feedback after updates, and confirmed when changes were live. Communication was clear and there were no major issues.

How did you publish the site and share it? I published using GitHub Pages and verified the live URL after pushing updates. I shared the link with the client and my junior developers so everyone could review the current version.

How was your experience managing the site with GitHub? It was a good learning experience. Pulling before editing and pushing updates helped avoid overwriting each otherâ€™s work, and GitHub Pages made it easy to verify changes.

Junior Developer Report

How did you communicate with your team leads? Any issues? I confirmed my page assignments early and provided updates when my pages were completed. The main issue I ran into was being blocked when a lead developer had not yet pushed the required HTML/CSS structure.

Did you know what was expected for each site? Yes. Each team had a site plan and homepage structure to follow, which made expectations clear.

Were you able to contribute? Yes. I completed and styled my assigned pages for one team and communicated completion to the lead developer.

Any GitHub issues? The biggest challenge was making sure I pulled the latest changes before editing to avoid conflicts. Once I followed that habit, it went smoothly.

General Experience and Reflections

Most challenging part: The most challenging part was coordinating timing and making sure everyone had pushed the files needed before the next person could work. I overcame it by communicating clearly and checking the repo often.

What did you enjoy most? I enjoyed seeing the site go from a plan to a real published project, and I liked learning how teamwork changes how you code and organize files.

What would you do differently? I would create a shared checklist at the start and confirm consistent naming (CSS files, folders, and links) so the team stays aligned.

Do you feel more confident now? Yes. This project made me more confident working with GitHub, communicating tasks, and contributing to a team project with real workflow steps like pull, commit, push, and publish.
